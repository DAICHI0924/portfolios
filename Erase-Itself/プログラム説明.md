# プログラム説明

## 開発環境・使用技術
### 開発環境
- **エンジン**: Unity 6000.3.11f1
- **言語**: C#
- **IDE**: Visual Studio Code

### 使用ライブラリ
- **R3**: リアクティブプログラミング
- **UniTask**: 非同期処理
- **LitMotion**: アニメーション処理（演出等）
- **DOTween**: アニメーション処理（UI等）
- **Cinemachine**: カメラ制御
- **KanKikuchi.AudioManager**: サウンド制御
- **unityroom.Api**: unityroomへのスコア送信

## コアシステム

### 1. ゲーム状態・イベント管理
ゲームの開始、プレイ中、クリア、ゲームオーバーといった状態の遷移と、それに伴う演出イベントを制御します。
- `InGameStateMachine` で各ステート（`GameStartState`, `PlayingState` 等）の遷移を制御
- `GameEventBus` と `EventListenerContainer` で状態変更に伴うイベント処理を一括管理

### 2. サービスロケータ
システム間の依存関係を整理し、機能の呼び出しや管理を容易にします。
- `ServiceLocator` でタイマー、プレイヤー、カメラなどのサービスインスタンスを管理
- `ServiceRegisterer` でシーン起動時に必要なサービスを自動登録

### 3. ゲーム演出・アニメーション制御
ゲーム開始時やクリア時のダイナミックなカメラワークと演出を非同期で実行します。
- `GameStartAnimator` でカメラ優先度切り替えやペンケースの開閉などを `LitMotion` シーケンスで実行
- `GameStartService` で演出中のUI表示切り替えやパラメータ初期化を実行

### 4. プレイヤー制御・物理システム
プレイヤー（消しゴム）の引っ張り射出や、走行中の回転、停止判定などの挙動を管理します。
- `PlayerBase` で速度や摩擦などのパラメータを定義し、物理挙動を設定
- `PlayerRotateManager` / `PlayerStopManager` で走行中の回転と停止を制御
- `LineDrawer` でドラッグ時の射出方向の予測線を描画

### 5. アイテム獲得・ギミックシステム
フィールド上に配置され、プレイヤーと衝突することで様々な効果を発動するギミック群です。
- `ICollectable` などのインターフェースを用いて衝突イベントを処理
- `ScoreUpItem` (スコア加算)、`VectorTurnAroundItem` (進行方向反転)、`AllPlayerMoveItem` (全プレイヤー同期移動) などを実装

### 6. UIとデータの連携（MVPパターン）
データ管理と画面描画を分離し、R3 を用いてデータの変更をUIに即座に反映します。
- `TimerPresenter` や `RemainAmountPresenter` でデータの更新を監視
- `TimerView` や `RemainAmountView` でテキスト更新やオブジェクトのスケール変更を描画
